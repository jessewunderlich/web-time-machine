## Tier A — Ship This Week (highest leverage, still unblocked)

### A.1 View Transitions on programmatic navigation _(1 hr, big feel)_

**Status:** Roadmap item 1.3, not shipped.

**Why now:** Cross-document View Transitions are **Baseline** as of late
2025 (Chrome 126, Edge 126, Safari 18.2 — ~85% of users). Same-document
view transitions became Baseline Newly Available on Oct 14, 2025 with
Firefox 144. Good progressive enhancement — unsupported browsers see
the current smooth-scroll.

**Implementation:** Wrap the scroll action in `ProgressBar.tsx` and
`KeyboardNav.tsx` with `document.startViewTransition(() => scrollTo(...))`
when available. In `globals.css`:

```css
@view-transition { navigation: auto; }
::view-transition-old(root),
::view-transition-new(root) {
  animation-duration: 0.4s;
  animation-timing-function: cubic-bezier(0.4, 0, 0.2, 1);
}
```

The **thematic payoff** is significant: a site about the evolution of web
tech using the newest web tech for its transitions. It's the kind of wink
Awwwards juries notice.

### A.2 Per-era dynamic OG images _(2–3 hr)_

**Status:** Roadmap item 1.5, not shipped.

**Why now:** Next.js 16 supports the `opengraph-image.tsx` file convention
with async `params`. This is the #1 growth lever still on the table —
right now a share of any era URL shows the same generic Web Time Machine
OG card. Era-specific OGs turn link shares into visual payloads that
match the destination.

**Implementation:** Use a dynamic route for shareability:

```
src/app/share/[era]/
  opengraph-image.tsx    # 1200×630 era-themed card
  page.tsx               # redirects to /#era-{era}
```

The `page.tsx` just 301s to `/#era-{era}` while serving the OG meta.
Can reuse the existing `_og/shared.tsx` ShareCard pattern with era
color/typography swap.

### A.3 Keyboard shortcut help dialog _(1 hr)_

**Status:** Roadmap item 2.5, not shipped.

**Why now:** `KeyboardNav`'s hint auto-hides after 3s and is
`aria-hidden`. Right now the arrow/1-7/Home/End shortcuts are effectively
a secret. A `?`-triggered modal with a keyboard legend fixes that — and
is discoverable because `?` is the universal "help" key since Gmail's
2007 interface.

Extra wink: render the help dialog **in the current era's style** —
Lynx-textual at 1991, GeoCities marquee at 1996, Flash popup at 2000,
etc. Converts a utility feature into a third easter egg layer.

## Tier B — Content Depth (half-day to a day each)

### B.1 Era 0: 1989–1990 (The Memo) _(3 hr)_

**Status:** Roadmap item 2.1, not shipped.

**What:** One era *before* the 1991 Lynx screen. CRT-green monospace
background showing Tim Berners-Lee's **March 1989 "Information
Management: A Proposal"** memo excerpts. Mike Sendall's handwritten
"vague but exciting" comment is the hook. Fade-in quote: "The World
Wide Web project has been surprisingly difficult to explain."

**Why:** Almost every visitor thinks the web started in 1991. The
1989–1990 prehistory (proposal accepted Oct 1990, first page written
Christmas Eve 1990) is genuinely unknown and gives the opening scroll
meaningful narrative momentum.

**Source content:** The memo is public (CERN, Creative Commons
attribution). Excerpt quotes verbatim; no fabrication risk.

### B.2 Era 8: 2026+ (The Agentic Era) _(1 day)_

**Status:** Roadmap item 2.2, not shipped.

**Why still urgent:** The site currently ends in "AI chatbots exist" —
which was frontier news 3+ years ago. Agentic interfaces (Claude Code,
Cursor, Codex, Arc Search, Dia, Perplexity Comet) are the *current*
frontier and genuinely a different design surface: tools-in-conversation,
streaming structured outputs, agent loops. Without this era, the site
starts dating badly by Q3 2026.

**Design:** Simulated agent conversation with real inline components —
the agent produces a flight search result card, a calendar picker, a
live code edit. Subtitle: "When the browser stopped being the canvas."

This is the single feature with the longest half-life.

### B.3 "Learn More" per-era drawer _(1 day)_

**Status:** Roadmap item 2.3, not shipped.

**What:** Each era gets a `[+] Learn More` button that slides up a
bottom sheet with:
- 3–5 paragraph long-form history
- Timeline of dated milestones
- Wayback Machine links to real period sites (working URLs)
- Tech-stack callouts (what HTML/CSS/JS features debuted)
- Attribution for any external sources

**Why:** Converts casual visitors to deep engagement without making the
casual tour heavier. Respects the "hey come look at this" moment while
offering real substance for anyone who wants it.

**Content-licensed:** cybercultural.com's period retrospectives can be
excerpted with attribution; Wikipedia for dated event lists; MDN for
the CSS/JS/HTML spec-ship dates.

### B.4 Real screenshots via Wayback Machine _(4 hr)_

**Status:** Roadmap item 2.4, tweaked.

**Update:** The original plan was webdesignmuseum.org screenshots — but
their licensing policy isn't publicly documented (would require direct
outreach). **Better path:** Use Wayback Machine iframes or static
screenshots from `web.archive.org/web/YYYYMMDD/<url>`.

Wayback Machine content is freely iframe-able with a rel="noopener"
sandbox — or we can generate our own static screenshots once via a
headless-Chrome script committed to the repo and never hit Wayback at
runtime.

Replace `HistoricalSites.tsx`'s CSS-art placeholders with real
screenshots. The CSS art stays as a fallback (e.g., when offline or
Wayback is slow) — classic progressive enhancement.

**Educational value:** "Real thing" >> "stylized representation" for
citability. Makes the site linkable from blog posts about web history
without the "those are made up" caveat.

## Tier C — Performance (mobile is the gap)

### C.1 Mobile performance pass _(4 hr)_

**Evidence:** Dev-mode Lighthouse mobile: Performance **79** (vs 98
desktop). LCP audit scoring **16/100**, TTI 53/100. Production will be
better due to minification but the gap is real.

**Known issues (from 2025 GSAP / iOS Safari research):**
1. `ScrollTrigger` pins jitter on iOS Safari when the address bar
   appears/disappears. **Fix:** Add
   `ScrollTrigger.config({ ignoreMobileResize: true })` in
   `ScrollProvider.tsx`.
2. Hydration cost on the cheap phones. **Fix:** Lazy-load Era 2021's
   GSAP bento grid via `next/dynamic` — it's the last era and
   below-the-fold for 95%+ of scroll sessions.
3. GSAP is imported in 9 separate components — the bundler can't
   tree-shake it well. **Fix:** One `src/lib/gsap.ts` that registers
   plugins and re-exports `gsap`/`ScrollTrigger`/`useGSAP`. Every
   component imports from there.
4. Legacy polyfills (`.at()`, `.flat()`, `.flatMap()`) still shipping
   in main bundle ~14KB. **Fix:** Tighten `browserslist` to modern
   browsers only; the site's aesthetic doesn't work below Chrome 90
   anyway.

**Target:** Mobile Perf 90+ in production.

### C.2 Bundle analyzer + tree-shake audit _(2 hr)_

**Evidence:** Lighthouse reports two chunks with >50% unused code (31KB
and 27KB). **Fix:** Install `@next/bundle-analyzer`, identify the
largest non-critical chunks, and either:
- Dynamic-import components used in one era only
- Remove unused CSS-in-JS generated selectors
- Split CSS modules per era (Next.js 16 should already do this)

### C.3 Preconnect + font optimization _(30 min)_

**Current:** `next/font` with local Inter/monospace. Lighthouse reports
render-blocking requests. **Fix:** Ensure `<link rel="preconnect">` for
any third-party font sources (shouldn't be any since we're using
next/font, but verify). Add `font-display: swap` if not already.

## Tier D — Dev Experience & Technical Hygiene

### D.1 Automated Lighthouse CI _(2 hr)_

**Why:** Every PR now does a manual Lighthouse round. GitHub Action
with `treosh/lighthouse-ci-action` runs on PR open + every push, fails
if perf < 90 or a11y < 100. Prevents silent regressions.

### D.2 Playwright visual regression _(3 hr)_

**Why:** Each pixel-accurate era is at risk of silent drift when CSS
utility classes get touched. Playwright's `toHaveScreenshot` at 3
viewports × 7 eras = 21 snapshots that fail noisily on any visual
change. Reviewer sees the diff inline in the GitHub PR.

### D.3 Migrate inline styles in `EraTransition` _(1 hr)_

`EraTransition.tsx` has extensive inline `style={{ ... }}` objects.
Works, but hard to theme or extract. Lift to a CSS module.

### D.4 Replace Web Audio in `SoundEngine` with actual synth lib _(half day)_

Current `SoundEngine.tsx` hand-rolls oscillators. Works but is fragile
(see the 2025 oscillator-leak fix). A tiny library like
`zzfx` (~6KB) handles lifecycle cleanly and lets you author sounds in
a standard format.

## Tier E — New Content Ideas

### E.1 "Seven Eras, Seven Songs" audio companion _(1 day)_

Lock a short authentic musical cue per era that plays (opt-in!) when
the era enters view:
- 1991: modem-handshake stutter
- 1996: chiptune MIDI clip (e.g., the Windows 95 startup rendered in
  8-bit)
- 2000: 10-second Linkin Park-era nu-metal riff (royalty-free
  substitute)
- 2005: MySpace auto-play indie rock sting
- 2010: Dubstep wobble
- 2015: LoFi beats to study to
- 2021: AI-generated ambient pad (Suno / Udio attributable track)

Gated behind the existing sound toggle. Zero-cost if someone doesn't
want it.

### E.2 Timeline scrubber view _(half day)_

Alternative navigation: horizontal timeline strip along the bottom on
desktop (optional, toggle off by default) showing all eras as a
miniature ribbon. Drag to scrub. Drop on an era → that era scrolls
into view with the ViewTransition from Tier A.1. Like YouTube's video
scrubber but for 35 years of the web.

Inspired by **Awwwards-featured** projects: Hg 25th Anniversary,
Enzo's Legacy (Ferrari), History in Motion (Friedrich Ebert
Foundation).

### E.3 "Today in Web History" daily card _(2 hr)_

Static JSON of "what happened on this day in web history" → API route
`/api/today` returns one JSON object. Footer of the site shows a
single-line card: "April 18 2003 — Apple launches iTunes Music Store."
Updates daily via ISR. Turns the site from a one-visit experience into
a "might check back" destination.

### E.4 Era achievements / completion tracking _(half day)_

LocalStorage-backed "you've visited 5 of 7 eras" indicator. Optional
sharable "I saw the whole web, 1989–2026" end-card with a little
signature SVG. Adds the completion-dopamine loop from the same family
as GitHub contribution graphs.

## Tier F — Growth & Distribution

### F.1 Product Hunt launch _(1 hr once ready)_

**Readiness checklist:**
- [ ] Era 8 (2026+) shipped (Tier B.2) — avoids "feels dated" comments
- [ ] Per-era OG images shipped (Tier A.2) — better shares in the

- [ ] View transitions shipped (Tier A.1) — makes the "wow" video

**Why now:** Educational/nostalgic/design-exemplar content does
consistently well on Product Hunt. Peer recognition from the web-dev
community is a strong lever for traffic + backlinks.

### F.2 Dev.to / Hacker News / r/webdev launch post _(2 hr writing)_

A "We built a time machine for the web, and here's what we learned"
write-up covering:
- What making period-accurate HTML/CSS looks like in 2026
- The a11y nightmare of recreating a GeoCities page that is still
  WCAG AA compliant
- The accuracy rigor (the fact-check round is a story in itself)

These platforms reward technical depth. Don't cross-post — stagger
over 2–3 weeks with slightly different angles.

### F.3 Embeddable era snippet _(1 day)_

`<iframe src="web-time-machine.com/embed/2000" width="800"
height="600" />` — lets bloggers embed a single era inside their own
web-history content. Creates natural backlinks and positions the site
as a canonical reference.

## Tier G — Big Swings (deliberate, not tomorrow)

### G.1 "Build Your Own Era" sandbox

Unchanged from previous roadmap — 2-day project, scope-risky, has to be
designed carefully so it doesn't become an open-ended Wix competitor.
Wait until the core is rock-solid.

### G.2 Multi-language pass

The Japanese web in the 2000s, the French Minitel-to-web transition,
the Chinese WeChat-is-the-web present — these are materially different
stories. **None of the current web-history sites do this.** Massive
differentiator if done right. Not a near-term item but worth banking
as the eventual "V2 killer feature."

### G.3 Curator mode

Add an admin-only flow where Jesse can propose new eras, events, or
corrections through a UI (backed by a JSON file in the repo via GitHub
API). Turns one-time content authoring into an ongoing editorial
habit. Could eventually open to community contributions with
moderation.

### G.4 Paid companion book / PDF

The long-form content in the Tier B.3 "Learn More" drawers could be
compiled into a $5 PDF or hardcover (Blurb / Amazon KDP). Not a
business — more a flywheel for the site's mission. Lightweight; tests
whether the audience wants depth.

## Things I Deliberately Did Not Recommend

- **Adding a database / CMS.** The repo is all-static, which is a
  superpower (Vercel free tier, infinite cache, no ops). Don't break that
  without a real reason.
- **Adding analytics.** There's Vercel Analytics already; more tracking
  adds page weight and a privacy-banner requirement without obvious
  upside for a personal project.
- **Rebuilding with a different framework.** Next 16 + React 19 + GSAP
  works beautifully here. Rewrite urges should be answered with
  "but what could we build in that time instead?"
- **Heavy SEO tooling (schema.org Timeline, JSON-LD events).** Tempting
  but low-ROI for a site whose best traffic path is Twitter/HN
  word-of-mouth, not search. Revisit if organic search becomes a
  measurable channel.
